# Contributing to Domus

Thank you for your interest in contributing to Domus! We welcome pull requests, bug reports, and design suggestions.

## Development Workflow

1. **Fork the repository** and create your branch from `main`.
2. **Ensure tests pass** before submitting.
3. **Format your code** using standard Elixir formatting (`mix format`).
4. **Keep UI changes aligned** with the high-end editorial design language. Avoid generic UI frameworks (e.g., Tailwind, Bootstrap).

## Reporting Bugs

Please use the GitHub Issue Tracker to report bugs. Include:
- Steps to reproduce the bug.
- Expected behavior vs actual behavior.
- Your OS and browser version.

## Design Philosophy

Domus strictly adheres to a "High-End Editorial" visual language:
- **Typography:** Playfair Display (headers) and Lato (body).
- **Colors:** `#F6F4F0` (background), `#1C1B1A` (text/lines).
- **Layout:** Thin borders, asymmetrical grids, no drop-shadows, no rounded corners.

Please ensure any frontend contributions strictly match this bespoke aesthetic.
